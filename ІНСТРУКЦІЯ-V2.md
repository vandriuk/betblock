## ✨ Що нового у версії 2.0?

### 1. **Дашборд з залишками по продукції**
- Показує залишки окремо для кожного типу продукції

- Видалення продукції

- Створення замовлень від клієнтів
- Статуси: Нове → В роботі → Готово → Виконано
- Примітки до замовлення
- Вибір продукції зі списку

### 5. **Експорт та бекап**
- **Експорт в Excel** - всі дані в один файл (.xlsx)
- **Бекап JSON** - повний бекап всієї бази даних

## 📋 Детальна інструкція Firebase

### КРОК 1: Створення проекту Firebase

1. Перейдіть на https://console.firebase.google.com
2. Натисніть "+ Додати проект"
3. Введіть назву: `production-v2`
4. Google Analytics - вимкніть (не потрібно)
5. Створити проект

### КРОК 2: Налаштування Authentication

1. У лівому меню: Build → Authentication
2. Get started
3. Sign-in method → Email/Password → Enable → Save

### КРОК 3: Налаштування Firestore

1. У лівому меню: Build → Firestore Database
2. Create database
3. Location: europe-west
4. Start in **test mode**
5. Enable

### КРОК 4: Налаштування правил Firestore

Перейдіть на вкладку **Rules** та замініть на:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    function isAuthenticated() {
      return request.auth != null;
    }
    
    function isAdmin() {
      return isAuthenticated() && 
        get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
    
    function isManager() {
      return isAuthenticated() && 
        get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'manager';
    }
    
    function canEdit() {
      return isAdmin() || isManager();
    }
    
    // Користувачі
    match /users/{userId} {
      allow read: if isAuthenticated();
      allow write: if isAdmin();
    }
    
    // Склад
    match /inventory/{doc} {
      allow read: if isAuthenticated();
      allow write: if canEdit();
    }
    
    // Продукція
    match /products/{doc} {
      allow read: if isAuthenticated();
      allow write: if canEdit();
    }
    
    // Виробництво
    match /production/{doc} {
      allow read: if isAuthenticated();
      allow create: if isAuthenticated();
      allow update, delete: if canEdit();
    }
    
    // Замовлення
    match /orders/{doc} {
      allow read: if isAuthenticated();
      allow write: if canEdit();
    }
    
    // Продажі
    match /sales/{doc} {
      allow read, write: if canEdit();
    }
    
    // Витрати
    match /expenses/{doc} {
      allow read, write: if canEdit();
    }
  }
}
```

Натисніть **Publish**

### КРОК 5: Додавання Web App

1. На головній сторінці проекту натисніть іконку **</>** (Web)
2. App nickname: `Production V2`
3. НЕ ставте галочку "Firebase Hosting"
4. Register app
5. **СКОПІЮЙТЕ firebaseConfig** - він виглядає так:

```javascript
const firebaseConfig = {
  apiKey: "AIzaSyC...",
  authDomain: "production-v2.firebaseapp.com",
  projectId: "production-v2",
  storageBucket: "production-v2.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};
```

### КРОК 6: Створення користувачів

#### 6.1 Додавання в Authentication
1. Authentication → Users → Add user

**Користувач 1 - Адміністратор:**
- Email: `admin@yourcompany.com`
- Password: `ваш_надійний_пароль`
- **Скопіюйте UID** (довга строка типу `xYz123...`)

**Користувач 2 - Менеджер:**
- Email: `manager@yourcompany.com`
- Password: `ваш_надійний_пароль`
- **Скопіюйте UID**

#### 6.2 Додавання ролей в Firestore
1. Firestore Database → Start collection
2. Collection ID: `users`
3. Document ID: **вставте UID адміністратора**
4. Додайте поля:
   - `email` (string): `admin@yourcompany.com`
   - `role` (string): `admin`
5. Save

Повторіть для менеджера з `role: manager`

### КРОК 7: Створення початкових даних

#### Колекція `products`
1. Start collection → Collection ID: `products`
2. Auto-ID
3. Поля:
   - `id` (number): `1`
   - `name` (string): `Шлакоблок`
   - `price` (number): `25`
4. Save

Додайте ще продукти:
- Бетоноблок: id=2, price=30

#### Колекція `inventory`
1. Start collection → Collection ID: `inventory`
2. Auto-ID
3. Поля:
   - `id` (number): `1`
   - `name` (string): `Цемент`
   - `unit` (string): `кг`
   - `quantity` (number): `0`
   - `minQuantity` (number): `500`
   - `price` (number): `5`
   - `notes` (string): ``
4. Save

Додайте ще матеріали (Пісок, Щебінь)

#### Порожні колекції
Створіть порожні колекції:
- `production`
- `orders`
- `sales`
- `expenses`

### КРОК 8: Оновлення коду

1. Відкрийте `index-firebase-v2.html` в текстовому редакторі
2. Знайдіть рядки (близько 21-27):

```javascript
const firebaseConfig = {
    apiKey: "ВАШ_API_KEY",
    authDomain: "ВАШ_PROJECT_ID.firebaseapp.com",
    ...
};
```

3. **Замініть** на ваші дані з Кроку 5
4. **Збережіть** файл

### КРОК 9: Деплой на Vercel

#### Через GitHub (рекомендовано)
1. Створіть репозиторій на GitHub
2. Завантажте `index-firebase-v2.html` як `index.html`
3. Перейдіть на https://vercel.com
4. Sign up with GitHub
5. New Project
6. Import вашого репозиторію
7. Deploy

**Готово!** Ваш сайт буде доступний за адресою `https://ваш-проект.vercel.app`


## 📖 Як користуватися

### 1. Дашборд
- Перегляд залишків по кожній продукції
- Моніторинг попереджень про склад
- Фінансова статистика

### 2. Склад
- Додавання матеріалів: кнопка "+ Додати матеріал"
- Редагування: кнопка "олівець" → змініть дані → кнопка "✓"
- Видалення: кнопка "корзина"
- Примітки: додайте будь-яку інформацію

### 3. Продукція
- Додайте нові типи продукції
- Встановіть ціну для кожного типу
- Редагуйте назву та ціну: кнопка "олівець" → змініть → "✓ Зберегти"
- Видаляйте: кнопка "корзина"

### 4. Виробництво
- Виберіть дату, зміну, продукцію
- Введіть кількість
- Додати

### 5. Замовлення
- Створіть замовлення від клієнта
- Змінюйте статус: Нове → В роботі → Готово → Виконано
- Додайте примітки

### 6. Продажі (адмін/менеджер)
- Виберіть продукцію зі списку
- Введіть кількість, ціну
- Відмітьте статус оплати

### 7. Витрати (адмін/менеджер)
- Виберіть категорію
- Введіть опис та суму

### 8. Експорт та бекап
- **Excel** - експортує всі дані в .xlsx файл
- **Бекап** - зберігає повний бекап у .json

## 💾 Бекап та відновлення

### Створення бекапу
1. Натисніть кнопку "Бекап" (фіолетова)
2. Збережеться файл `backup_2024-XX-XX.json`

### Відновлення з бекапу
Якщо потрібно відновити дані:
1. Відкрийте Firebase Console → Firestore
2. Import/Export → Import
3. Виберіть ваш backup файл

### Експорт в Excel
1. Натисніть кнопку "Excel" (зелена)
2. Збережеться файл `backup_2024-XX-XX.xlsx` з листами:
   - Склад
   - Продукція
   - Виробництво
   - Замовлення
   - Продажі
   - Витрати

## ❓ Вирішення проблем

### Помилка: "Firebase не налаштовано"
**Рішення:** Перевірте чи правильно скопіювали firebaseConfig

### Помилка: "Permission denied"
**Рішення:** Перевірте правила Firestore (Крок 4)

### Не можу увійти
**Рішення:**
1. Перевірте чи створено користувача в Authentication
2. Перевірте чи додано роль у Firestore → users

### Не працює експорт
**Рішення:** Перевірте чи підключена бібліотека XLSX (рядок 14 в коді)
